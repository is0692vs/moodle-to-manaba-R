# Phase 1 MVP テスト手順

## セットアップ

1. **Chrome拡張機能をインストール**
   ```
   chrome://extensions/ にアクセス
   → 「デベロッパーモード」をON
   → 「パッケージ化されていない拡張機能を読み込む」
   → このディレクトリ（moodle-to-manaba-extension）を選択
   ```

2. **Moodleにログイン**
   - `https://moodle.ritsumei.ac.jp/my/` または
   - `https://lms.ritsumei.ac.jp/my/` にアクセス

## テストシナリオ

### 基本動作確認

1. **ページ読み込み時の動作**
   - [ ] ページアクセス時にコースカードが動的に読み込まれる
   - [ ] 元のMoodleコースカードが非表示になる
   - [ ] 「manabaスタイルの時間割を生成しています…」メッセージが表示される
   - [ ] manabaスタイルのテーブルが表示される

2. **テーブル構造の確認**
   - [ ] 7時限×6曜日（月〜土）のテーブルが生成される
   - [ ] ヘッダー行に曜日が正しく表示される
   - [ ] 左端の列に時限番号（1〜7）が表示される

3. **コース情報の表示**
   - [ ] 各コースが正しい曜日・時限に配置される
   - [ ] コース名がリンクとして表示される
   - [ ] 教室情報が「曜日時限:教室名」形式で表示される

### 高度なテスト

4. **範囲時限の確認**
   - [ ] `金1(1-2)`のような範囲時限が複数セルに展開される
   - [ ] 同じコースが連続する時限に表示される

5. **複数コースの処理**
   - [ ] 同じ曜日・時限に複数コースがある場合、両方表示される
   - [ ] コースが縦にスタックして表示される

6. **エラーケースの処理**
   - [ ] 時間割情報がないコースは表示されない
   - [ ] ネットワークエラー時にエラーメッセージが表示される
   - [ ] ページ全体が壊れない

### スタイリング確認

7. **見た目の確認**
   - [ ] manabaに近いテーブルデザイン
   - [ ] ホバー効果が動作する
   - [ ] 教室情報が適切にスタイリングされる
   - [ ] レスポンシブに表示される

## 期待される動作

### 正常ケース
```
元のMoodleコースカード → 非表示
↓
「時間割を生成しています…」メッセージ
↓
manabaスタイルの7×6時間割テーブル表示
```

### エラーケース
```
読み込みエラー → 「時間割の生成中にエラーが発生しました」メッセージ
時間割情報なし → 「時間割情報を含むコースが見つかりませんでした」メッセージ
```

## デバッグ情報

**開発者ツールコンソールで確認:**
- `[M2M]` プレフィックスのログメッセージ
- パースされたスケジュール情報
- フェッチエラーの詳細

**DOM要素の確認:**
- `#manaba-timetable-wrapper` がコンテナとして存在
- `table.stdlist.manaba-timetable` が時間割テーブル
- 元の `[data-region="course-view-content"]` が `display: none`

## トラブルシューティング

**テーブルが表示されない場合:**
1. コンソールエラーを確認
2. ネットワークタブでfetchエラーを確認  
3. コースに時間割情報が含まれているか確認

**スタイルが適用されない場合:**
1. styles.cssが正しく読み込まれているか確認
2. 他のCSSとの競合がないか確認
